# BLE Scanner Pro — Electron + Next.js + Noble (Windows)

> **Project**: Full-featured BLE scanning, analysis, and sniffing desktop application for Windows.  
> **Platform**: Galaxy Book 6 Pro (Windows 11, built-in BLE adapter)  
> **Why Electron + Noble**: `requestLEScan()` is NOT supported on Windows (Chromium #897312). The only way to access raw BLE advertising packets, RSSI, and full GATT exploration on Windows is through native Node.js bindings.

## Features

### 1. Real-Time BLE Scanner
- Start/stop scanning with configurable duration
- Live device table: name, address, RSSI, advertising data
- Color-coded RSSI strength indicator (strong/medium/weak)
- Device deduplication by MAC address with timestamp tracking
- Per-device scan count and first-seen / last-seen timestamps
- Filter by name, UUID, manufacturer ID, address type
- Sort by RSSI, name, or last seen
- Export scan results (JSON, CSV)

### 2. Advertising Data Analyzer
- Raw advertising data in hexadecimal with ASCII decode
- Structured parsing of BLE AD structures:
  - Flags, Complete/Incomplete Local Name, Shortened/Complete Name
  - Service UUIDs (16-bit, 32-bit, 128-bit)
  - Service Data with UUID + payload
  - Manufacturer Specific Data (company ID lookup from Bluetooth SIG registry)
  - TX Power Level
  - Appearance
  - Solicited Services
- Visual hex editor view with byte highlighting

### 3. RSSI Signal Tracking
- Real-time RSSI per device with history chart (line graph)
- RSSI trend: improving / stable / deteriorating
- Signal-to-noise estimation with visual meters
- RSSI threshold alerts (configurable)

### 4. GATT Explorer (Connected Devices)
- Connect to any discovered BLE device
- Full service discovery with tree view:
  - Services → Characteristics → Descriptors
- Read/write characteristic values (hex + text)
- Subscribe to characteristic notifications/indications
- Descriptor read/write (CCCD, etc.)
- MTU negotiation display
- Connection parameters display (interval, latency, timeout)

### 5. Sniffing Tools
- Advertisement packet capture with timestamp, interval analysis
- Advertising interval estimation per device
- Packet frequency heatmap
- Advertisement type classification (scan response, connectable, undirected, directed)
- Capture session save/load (JSON export)
- Device fingerprinting from advertising patterns

### 6. Device Management
- Saved/favorite devices list
- Device profiles: name aliases, notes, custom service lists
- Connection history
- Quick-connect to saved devices

### 7. Analysis Dashboard
- Summary statistics: devices discovered, active connections, scan duration
- Network topology visualization (devices as nodes)
- Service UUID distribution chart
- Manufacturer data breakdown
- Real-time event log with severity levels

### 8. UI/UX
- Dark mode as default (light mode toggle)
- Responsive layout resizable in Electron window
- Sidebar navigation with collapsible panels
- Keyboard shortcuts for scan control
- Toast notifications for connection events
- Drag-and-drop session file import

## Important Considerations

### Windows BLE Limitations
- Noble on Windows uses Windows Bluetooth APIs (BluetoothAPI.h)
- Requires Windows 10+ (Galaxy Book 6 Pro = Win 11 ✅)
- BLE adapter must not be paired/bound by Windows for optimal scanning
- User must allow Bluetooth access (Windows Core Bluetooth permission)

### Security
- IPC with `contextBridge` (NOT `nodeIntegration: true`)
- No unsandboxed renderer access to BLE APIs
- All BLE operations go through validated IPC channels

### Performance
- Device deduplication to prevent memory leaks during long scans
- Debounced IPC events (not every raw advertisement packet)
- Virtualized table for 1000+ devices
- Automatic scan cleanup after session

### Advertising Data AD Type Reference
| Type | Code | Description |
|---|---|---|
| Flags | 0x01 | LE discovery procedures |
| Incomplete 16-bit UUID | 0x02 | List of service UUIDs |
| Complete 16-bit UUID | 0x03 | All service UUIDs |
| Incomplete Name | 0x08 | Shortened local name |
| Complete Name | 0x09 | Full device name |
| TX Power | 0x0A | TX power level |
| 16-bit UUID w/ Data | 0x16 | Service data |
| 32-bit UUID w/ Data | 0x20 | Service data |
| 128-bit UUID w/ Data | 0x24 | Service data |
| Manufacturer Data | 0xFF | Company-specific data |

## Chrome Alternative (Future / macOS)

For macOS users who prefer browser-only access:
- Enable `chrome://flags/#enable-experimental-web-platform-features`
- `requestLEScan()` available on Chrome 79+ macOS/Android
- Limited to granted devices with `listenOnlyGrantedDevices: true`
- NO sniffing of ungranted devices, NO raw packet capture
- Recommended as secondary browser mode only
